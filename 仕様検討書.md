# 日本年金機構 電子通知書 変換ツール 仕様検討書

## 1. 前提の整理

### 1.1 nagare.pdf の内容

| 項目 | 内容 |
|------|------|
| 文書名 | ご利用までのながれ（手順書）第3.00版 |
| 目的 | 被保険者単位の通知書 → **届出（到達番号）単位**の通知書形式へ変換 |
| 変換方法 | 通知書フォルダ内の.xslを RenkiStylesheet の.xslで**置き換える** |
| 表示 | XMLファイルをダブルクリック → ブラウザで一覧表形式表示 |

### 1.2 変換の仕組み

```
┌─────────────────────────────────────────────────────────────────┐
│  通知書フォルダ（変換前）                                          │
│  ├── 7120002.xml  ← 日本年金機構から届いた電子通知書               │
│  └── 7120002.xsl  ← 被保険者単位のスタイル（1人1枚形式）           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │  .xsl を RenkiStylesheet の .xsl で置換
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  変換フォルダ（変換後）                                            │
│  ├── 7120002.xml                                                  │
│  └── 7120002.xsl  ← 届出単位のスタイル（一覧表形式）              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │  XML をダブルクリック
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  ブラウザ表示：届出（到達番号）単位の一覧表形式                     │
│  （nagare.pdf 最終ページの「表示イメージ」）                        │
└─────────────────────────────────────────────────────────────────┘
```

## 2. ご要望の整理

| # | ご要望 | 解釈 |
|---|--------|------|
| 1 | 変換フォルダ内のファイルを表示イメージのように表示 | XML + RenkiStylesheet の XSL で HTML を生成し、一覧表形式で表示 |
| 2 | 出力させる | 表示結果を **PDF 等のファイルとして出力** |
| 3 | 他者に連携できるツール | Cursor に依存せず、**スタンドアロンで配布・利用可能**な形態 |

## 3. 仕様の満たし方（実現可能性）

### 3.1 技術的な実現性

| 処理 | 実現方法 | 難易度 |
|------|----------|--------|
| XML + XSL → HTML | XSLTProcessor（ブラウザ）、xslt（Node.js）、lxml（Python） | ◎ 容易 |
| HTML → PDF | Puppeteer / Playwright、weasyprint、ブラウザ印刷 | ◎ 容易 |
| スタンドアロンツール | 単一HTML + JS、Electron、CLI、PWA | ◎ 容易 |

**結論：技術的には十分実現可能です。**

### 3.2 ツール形態の候補

| 形態 | メリット | デメリット | 配布のしやすさ |
|------|----------|------------|----------------|
| **A. 単一HTMLファイル** | サーバー不要、ダブルクリックで起動、配布が簡単 | PDF出力はブラウザの印刷機能に依存 | ◎ |
| **B. Electron アプリ** | デスクトップアプリとして完結、PDF自動出力可能 | ビルド・配布がやや重い | ○ |
| **C. コマンドラインツール（Node/Python）** | 自動化・バッチ処理に強い | 実行環境のインストールが必要 | △ |
| **D. Webアプリ（ローカル実行）** | 見た目が良い、機能拡張しやすい | 簡易サーバー起動が必要な場合あり | ○ |

**推奨：まず A（単一HTML）で実装し、必要に応じて B や C に拡張**

## 4. 注意事項・制約（nagare.pdf より）

| 項目 | 内容 |
|------|------|
| 外字 | **外字を含む電子通知書は一覧表形式に変換できない** |
| ファイル名 | XSL のファイル名を変更しないこと（既存の .xsl と同名である必要あり） |
| 元データ | 変換前に通知書フォルダを**複製**してから .xsl を置換すること（元に戻せなくなるため） |

---

## 5. 不足している情報・確認事項

| # | 項目 | 説明 |
|---|------|------|
| 1 | **変換フォルダの実体** | 現在ワークスペースに XML ファイル・変換フォルダが見当たりません。サンプル XML の有無、またはフォルダ構成の想定を教えてください。 |
| 2 | **表示イメージの詳細** | nagare.pdf 最終ページの「（表示イメージ）」は PDF 内に画像として埋め込まれている可能性があります。具体的なレイアウト（表の列・行構成など）の指定があれば、より正確に再現できます。 |
| 3 | **出力形式の優先** | PDF 必須か、HTML のままでもよいか。複数 XML をまとめて 1 PDF にするか、1 XML = 1 PDF か。 |
| 4 | **利用者環境** | 主な利用者は Windows / Mac / 両方？ ブラウザは何を使用する想定か。 |
